# Deploying Urbis (production)

Urbis is split into three hosted services. Local dev uses the same code paths; production only changes **environment variables** and **URLs**.

```
┌─────────────┐     HTTPS      ┌─────────────┐     ┌──────────────┐
│   Vercel    │ ─────────────► │   Render    │ ──► │ MongoDB Atlas│
│  (React)    │   API calls    │  (FastAPI)  │     │  (database)  │
└─────────────┘                └──────┬──────┘     └──────────────┘
                                      │
                    ┌─────────────────┼─────────────────┐
                    ▼                 ▼                 ▼
              Cloudinary         lemma.work          Gmail OAuth
              (images)           (AI agents)         (send email)
```

## Prerequisites

| Service | Purpose | Sign up |
|---------|---------|---------|
| **MongoDB Atlas** | Primary database | [mongodb.com/atlas](https://www.mongodb.com/atlas) |
| **Render** | API hosting | [render.com](https://render.com) |
| **Vercel** | Frontend hosting | [vercel.com](https://vercel.com) |
| **Cloudinary** | Image storage (required on Render) | [cloudinary.com](https://cloudinary.com) |
| **Lemma** | AI agents (optional) | [lemma.work](https://lemma.work) |
| **Google Cloud** | OAuth + Gmail send | [console.cloud.google.com](https://console.cloud.google.com) |

---

## 1. MongoDB Atlas

1. Create a free cluster.
2. Database Access → create user with read/write.
3. Network Access → allow `0.0.0.0/0` (or restrict to Render IPs later).
4. Connect → copy connection string:
   ```
   mongodb+srv://USER:PASS@cluster.mongodb.net/urbis?retryWrites=true&w=majority
   ```

---

## 2. Deploy API (Render)

### Option A — Blueprint (recommended)

1. Push this repo to GitHub.
2. Render → **New** → **Blueprint** → connect repo (`render.yaml` is auto-detected).
3. Set **sync: false** variables in the dashboard (see checklist below).

### Option B — Docker on any host

```bash
cp .env.production.example .env
# Edit .env with real values
docker compose -f docker-compose.prod.yml up -d --build
```

### Render environment variables

Copy from [`../.env.production.example`](../.env.production.example):

| Variable | Example | Required |
|----------|---------|----------|
| `MONGODB_URL` | `mongodb+srv://...` | Yes |
| `SESSION_SECRET` | auto-generated by Render | Yes |
| `API_BASE_URL` | `https://urbis-api.onrender.com` | Yes |
| `FRONTEND_URL` | `https://urbis.vercel.app` | Yes |
| `CORS_ORIGINS` | same as frontend URL | Yes |
| `GOOGLE_CLIENT_ID` / `SECRET` | from Google Cloud | Yes (for auth) |
| `GOOGLE_REDIRECT_URI` | `https://<api>/api/auth/google/callback` | Yes |
| `CLOUDINARY_*` | three values from dashboard | Yes |
| `LEMMA_REFRESH_TOKEN` | from `lemma auth login` | Optional |
| `LEMMA_POD_ID` / `LEMMA_ORG_ID` | from Lemma pod | Optional |
| `SMTP_*` | Brevo fallback | Optional |

**Lemma on Render:** run locally once:

```bash
backend/.venv/bin/lemma auth login
./scripts/sync-lemma-env.sh
```

Copy `LEMMA_REFRESH_TOKEN`, `LEMMA_POD_ID`, `LEMMA_ORG_ID` into Render. Do **not** rely on `~/.lemma/config.json` on the server.

### Verify API

```bash
curl https://<your-api>.onrender.com/api/health/live
curl https://<your-api>.onrender.com/api/health
```

## 3. Deploy frontend (Vercel)

1. Import GitHub repo → set **Root Directory** to `frontend`.
2. Framework: Vite (auto-detected).
3. Environment variable:
   ```
   VITE_API_URL=https://<your-render-api>.onrender.com
   ```
4. Deploy.

`frontend/vercel.json` handles SPA routing.

---

## 4. Google OAuth (production)

In [Google Cloud Console](https://console.cloud.google.com/apis/credentials):

1. OAuth client → **Authorized JavaScript origins**: `https://<vercel-domain>`
2. **Authorized redirect URIs**: `https://<render-api>/api/auth/google/callback`
3. Publish OAuth consent screen (Testing → Production when ready).

Set on Render:

```
GOOGLE_REDIRECT_URI=https://<render-api>/api/auth/google/callback
FRONTEND_URL=https://<vercel-domain>
COOKIE_SECURE=true
COOKIE_SAMESITE=none
```

## 5. Pre-deploy checklist

Run locally before pushing:

```bash
chmod +x scripts/check-deploy-ready.sh
ENVIRONMENT=production \
MONGODB_URL='mongodb+srv://...' \
API_BASE_URL='https://your-api.onrender.com' \
FRONTEND_URL='https://your-app.vercel.app' \
GOOGLE_REDIRECT_URI='https://your-api.onrender.com/api/auth/google/callback' \
./scripts/check-deploy-ready.sh
```

## Local vs production

| | Local | Production |
|---|-------|------------|
| MongoDB | `mongodb://localhost:27017` | MongoDB Atlas |
| API | `http://localhost:8000` | Render HTTPS URL |
| Frontend | `http://localhost:5173` (Vite proxy) | Vercel + `VITE_API_URL` |
| Images | Cloudinary or local `uploads/` | **Cloudinary required** |
| Lemma tokens | `lemma auth login` + sync script | `LEMMA_REFRESH_TOKEN` in Render |
| Cookies | `COOKIE_SAMESITE=lax` | `none` + `SECURE=true` |
| Email test mode | `DEMO_EMAIL_REDIRECT=true` | **must be `false`** |

### Local without Docker

```bash
./scripts/setup.sh          # once
./scripts/run-local.sh      # API + frontend
```

### Local with Docker

```bash
./scripts/start.sh
cd frontend && npm run dev
```

## Troubleshooting

| Symptom | Fix |
|---------|-----|
| OAuth redirect mismatch | `GOOGLE_REDIRECT_URI` must exactly match Google Console |
| Login works locally, not prod | Set `COOKIE_SAMESITE=none`, `COOKIE_SECURE=true` |
| Images disappear after restart | Configure Cloudinary on Render |
| Lemma agents fail | Refresh `LEMMA_REFRESH_TOKEN`; only one API instance should refresh |
| CORS errors | Add Vercel URL to `CORS_ORIGINS` and `FRONTEND_URL` |
| API won't start | Check Render logs — production validates config on boot |
